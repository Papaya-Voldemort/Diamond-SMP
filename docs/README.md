# Documentation Map

This repository now documents the implemented plugin, not the planned plugin.

## Recommended Reading Order

1. [README](/Volumes/External%20Home/Kids%20Home/dev/Diamond-SMP/README.md) for the full product overview.
2. [Configuration Guide](/Volumes/External%20Home/Kids%20Home/dev/Diamond-SMP/docs/guides/configuration-guide.md) for operator setup.
3. [Feature Index](/Volumes/External%20Home/Kids%20Home/dev/Diamond-SMP/docs/features/README.md) for system-by-system behavior.
4. `architecture/` for module boundaries and lifecycle rules.
5. `guides/` for contributor workflow, release process, and build instructions.

## Documentation Intent

- `features/` explains what is implemented, how to use it, and what is intentionally not present.
- `guides/` explains how to configure, run, and maintain the plugin.
- `architecture/` explains how the multi-module project is wired.
- `templates/` remains available for future ADRs, specs, and tasks.
