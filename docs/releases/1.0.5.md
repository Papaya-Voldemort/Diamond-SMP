# Diamond SMP 1.0.5

Diamond SMP `1.0.5` fixes the restricted enchant flow so illegal survival enchants are normalized back to the configured cap instead of breaking anvils or leaking through villager books.

## Highlights

- Downgrades normal `Protection IV` to `Protection III`
- Downgrades normal `Sharpness V` to `Sharpness IV`
- Sanitizes both direct enchants and stored book enchants
- Cleans up villager trades so normal villagers cannot continue selling illegal enchant books
- Preserves event-exclusive god items as the intended bypass path

## Runtime

- Minecraft `1.21.11`
- Paper `1.21.11`
- Java `21`

## Artifact

- `plugin-bootstrap/build/libs/Diamond-SMP-1.0.5.jar`

## Validation

- `./gradlew :platform-paper:compileJava`
- `./gradlew :plugin-bootstrap:shadowJar`
- Verified restricted enchant outputs are capped instead of blocked in the plugin enforcement layer
